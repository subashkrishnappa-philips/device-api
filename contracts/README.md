# contracts/

This directory is the **shared contract artefacts** folder for the DeviceApi ecosystem.

## Contents

| File / Folder | Description |
|---|---|
| `consumers.json` | Registry of all downstream consumer teams and their contact details. |
| `DeviceApi-Consumer-DeviceApi.json` | Pact file generated by consumer tests (runtime artefact). |
| `swagger-baseline.json` | Committed snapshot of the DeviceApi swagger used for drift detection. |
| `consumer-notification.md` | Auto-generated notification report (produced by SwaggerPactGenerator). |

## Lifecycle

```
Developer adds/changes endpoint in src/DeviceApi
         ↓
run-pact-tests.ps1 runs SwaggerPactGenerator
         ↓
Generator creates stubs in tests/DeviceApi.Consumer.Tests/Generated/
Generator writes contracts/consumer-notification.md
         ↓
SwaggerContractValidatorTests  ← FAILS (swagger drift detected)
    prints notification with team names, emails, Slack channels
         ↓
Consumer team fills in Generated/ stub → moves to Contracts/
Consumer team re-runs consumer tests → new pact file written
         ↓
run-pact-tests.ps1 --update-baseline  ← resets baseline
         ↓
All tests pass ✓
```

## Adding a New Consumer

Edit `consumers.json` and add an entry:

```json
{
  "name": "MyApp-Consumer",
  "team": "My Application Team",
  "contact": "my-team@example.com",
  "slack": "#my-team-platform",
  "pactFile": "MyApp-Consumer-DeviceApi.json",
  "notes": "Optional free-text notes."
}
```

The consumer team must:
1. Create their own consumer test project.
2. Point it at this `contracts/` directory for pact output.
3. Register themselves in `consumers.json` so they get notified on contract changes.
